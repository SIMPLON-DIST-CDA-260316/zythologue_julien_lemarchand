## Consignes

Pour le projet que vous devez corriger :

1. Clonez le repo du candidat.
2. Lisez le `README.md`.
3. Vérifiez que le projet est compréhensible et facile à prendre en main.
4. Lancez l’environnement avec Docker.
5. Connectez DBeaver à la base PostgreSQL du Docker.
6. Testez les fichiers SQL présents dans le repo.
7. Notez vos remarques dans un fichier Markdown sur votre repo d’origine.

## Fichier de correction attendu

Dans votre propre repo, ajoutez un fichier nommé :

`correction-pair.md`

Ce fichier doit contenir les sections suivantes :

- Correcteur
- Projet corrigé
- Prise en main du projet
- Docker
- PostgreSQL / DBeaver
- Fichiers SQL
- Remarques générales

## Points à vérifier

### Prise en main du projet

- Le README est-il clair ?
- Les étapes d’installation sont-elles compréhensibles ?
- Les commandes Docker sont-elles présentes et fonctionnelles ?

### Docker

- Le projet se lance-t-il correctement ?
- Les services attendus sont-ils bien disponibles ?
- Y a-t-il des erreurs ou des étapes manquantes ?

### PostgreSQL / DBeaver

- La connexion à PostgreSQL fonctionne-t-elle ?
- Les informations de connexion sont-elles faciles à trouver ?
- La base est-elle correctement initialisée ?

### Fichiers SQL

- Les fichiers SQL s’exécutent-ils correctement ?
- Les tables sont-elles créées ?
- Les données sont-elles cohérentes ?
- Y a-t-il des erreurs SQL ?

### Remarques générales

- Points positifs
- Points à améliorer
- Conseils pour rendre le projet plus clair ou plus robuste

## Important

La correction doit être factuelle, bienveillante et utile.

Le but n’est pas de juger la personne, mais de vérifier si un autre développeur peut reprendre le projet facilement à partir du README, de Docker et des fichiers SQL.
